# 桌面宠物

使用 OpenAI Sora 生成的真实动画打造的 Electron 桌面宠物。下载后由 ffmpeg 一次性烘焙成带真透明通道的 WebM，渲染层直接播放透明视频，桌面上宠物背景完全透明、无绿色残留，支持本地离线使用。

## 工作原理

1. **动画生成**：上传宠物参考照片 → 调用 GPT Image（gpt-image-1.5）生成标准坐姿参考 → 调用 Sora 生成三段动画（待机、站起、睡觉）
2. **透明烘焙**：Sora 输出 720x1280、#00ff00 纯绿色背景的 MP4；下载后用 ffmpeg 抠掉绿幕并烘焙成 **带透明通道的 WebM（VP9, yuva420p）**，一次处理、长期复用，避免在压缩视频上实时抠绿幕留下的绿色边缘
3. **切换无跳变**：烘焙时给每段动画首尾各定格约 0.25 秒的标准坐姿帧（ffmpeg `tpad`），渲染层切换动作时再叠加 150ms 交叉淡化，多段动画衔接平滑、无空白帧
4. **本地渲染**：渲染层用 WebGL 直接采样 WebM 自带的 alpha 通道（非预乘 alpha），实时绘制到透明无边框窗口
5. **离线使用**：烘焙后的透明 WebM 保存到本机，重启应用时离线加载，无需重新生成

## 运行

先安装依赖：

```bash
npm install
```

> 依赖中包含 `ffmpeg-static`，会自带 ffmpeg 二进制用于透明烘焙，**用户无需单独安装 ffmpeg**。

启动项目：

```bash
npm start
```

应用启动后会打开宠物创建界面。若已生成过宠物，会直接显示桌面宠物窗口（使用本地保存的透明动画）。

直接打开宠物创建窗口：

```bash
npm run settings
```

## 构建 macOS App

当前本机测试包面向 Apple Silicon Mac（M1/M2/M3/M4 等），无需开发者证书：

```bash
npm install
npm run app:mac:clean
```

构建结果：

```text
release/mac-arm64/PetMo.app
```

可以直接在 Finder 中双击使用，或拖到 `/Applications` 文件夹。若 macOS 首次启动时提示安全确认，可右键 `PetMo.app` 选择“打开”。

当前构建是本机测试用未签名版本。向其他用户正式发布前，需要增加 Apple Developer ID 签名、公证，并生成 DMG 安装镜像。

## 当前功能

### 桌面宠物窗口

- ✅ 透明无边框悬浮窗口（动态大小调整）
- ✅ 始终置顶显示

- ✅ 点击宠物立即切换到下一个随机动作
- ✅ 右键宠物退出应用
- ✅ 直接播放带透明通道的 WebM，桌面背景完全透明、无绿色残留
- ✅ 动作切换时首尾定格标准坐姿 + 150ms 交叉淡化，衔接平滑无跳变
- ✅ 右上角设置按钮打开宠物创建窗口

### 离线使用和本地持久化

- ✅ 所有生成的透明 WebM 动画保存到本机（`~/.config/desktop-pet/`）
- ✅ 重启应用时自动加载本地动画，无需 API 调用
- ✅ 支持重新生成（新生成完成前使用旧动画）
- ✅ 可随时切换不同的宠物版本

## 创建自己的宠物

点击桌面宠物右上角的设置按钮，打开宠物创建窗口：

### 上传宠物参考照片

- 上传 **单张正面照片** 作为宠物参考（PNG、JPG 或 WebP）
  - 或继续支持上传四张不同视角照片（旧流程兼容）
- API Key 由 Electron `safeStorage` 加密后保存在本机

### 图片模型已锁定

- 参考图固定使用 **gpt-image-1.5**，设置页内该字段已锁定，用户无需也无法修改
- 这样可保证生成的标准坐姿参考图风格一致，避免误选不兼容的模型

### 获取并保存 API Key

- 设置页内置 **中英文可切换** 的”如何获取 API Key”说明书，按指引到 [platform.openai.com/api-keys](https://platform.openai.com/api-keys) 创建密钥即可
- API Key 由 Electron `safeStorage` **加密保存在本机**，不会上传到任何服务器
- ⚠️ 请勿泄漏你的 API Key：任何人拿到密钥都能消耗你的额度

### 自动生成流程

点击 **生成** 后应用会自动执行以下步骤（显示进度）：

1. **生成标准坐姿参考图**（使用 GPT Image, gpt-image-1.5）
   - 基于上传的照片生成统一视角和光线的标准坐姿参考
2. **生成待机动画**（Sora, 4秒）
   - 宠物保持标准坐姿，做自然动作
3. **生成站起动画**（Sora, 8秒）
   - 宠物从坐姿站起并重新坐下
4. **生成睡觉动画**（Sora, 8秒）
   - 宠物从坐姿缓慢趴下并坐回
5. **透明烘焙 + 本地保存**
   - 每段视频用 ffmpeg 烘焙成透明 WebM（首尾定格标准坐姿），所有资源保存到本机，完成后自动使用新宠物

### 失败重试

- 如果任何一个动作生成失败，支持单独重试该动作
- 重新生成时，已成功的动作不会重新生成
- 生成过程中继续使用旧宠物（若存在）

### 再次启动

- 若无宠物数据，启动时打开宠物创建窗口
- 若有宠物数据，启动时直接显示桌面宠物
- 点击设置按钮可随时打开创建窗口重新生成或切换版本

## API 权限和费用说明

### 必需的 API 权限

- **OpenAI API Key** 必填
- **Sora Videos API**：需要在 [platform.openai.com](https://platform.openai.com) 申请访问权限
  - 当前 Sora 处于受限测试阶段，可在 OpenAI 官网申请加入等待列表
- **GPT Image（gpt-image-1.5）**：用于生成标准坐姿参考图

### 费用估算

每次生成一个新宠物需要调用：

- 1 次 GPT Image（生成标准坐姿参考图）
- 3 次 Sora Videos API（待机、站起、睡觉三段视频）

根据 OpenAI 官方价格（2026年6月参考）：
- GPT Image（gpt-image-1.5）：约 $0.08（用于生成参考图）
- Sora Videos API：每段约 $0.10~0.15

**总费用估算**：每个新宠物约 $0.40~0.53（约 RMB 2.8~3.8）

### 降低成本的方式

- **本地缓存**：生成后的宠物离线使用，无需重新生成
- **增量生成**：重新生成时可保留已成功的动画，只重试失败的部分
- **共享宠物**：可将生成好的宠物数据文件分享给他人使用

## 故障排除

### API Key 配置问题

**问题**：显示”API Key 未设置”或”API Key 无效”

**解决**：
- 确保 API Key 正确复制自 [platform.openai.com/api-keys](https://platform.openai.com/api-keys)
- Key 首次设置后会加密保存，无需重复输入
- 如需更换 Key，点击”清除 API Key”后重新输入

### 权限错误

**问题**：报错”You exceeded your current quota”或”This API is not available”

**解决**：
- 检查 OpenAI 账户是否有足够余额
- 检查是否已申请 Sora Videos API 权限
- 如未获得权限，访问 OpenAI 官网提交申请

### 网络超时和重试

**问题**：生成中断或显示”网络超时”

**工作机制**：
- Sora 生成通常需要 30 秒 ~ 2 分钟
- 应用会自动重试最多 3 次（等待时间分别为 10秒、20秒、40秒）
- 若仍失败，会提示”生成失败”，支持点击重试单个动作

### 单个动作失败

**问题**：其中某个动作生成失败，其他动作已完成

**解决**：
- 界面会显示哪个动作失败
- 点击”重试”按钮仅重新生成失败的动作
- 不会重新生成已成功的部分

### 使用旧资源 / 回退

**问题**：生成中想取消或生成出现问题

**工作机制**：
- 生成过程中继续使用旧宠物（若存在）
- 生成完成后才会切换到新宠物
- 新宠物完全准备好前，旧宠物数据不会删除

## 技术栈

- **Electron 43**：桌面应用容器，透明窗口支持
- **OpenAI Node SDK 6.44**：Sora Videos、GPT Image（gpt-image-1.5）API 调用
- **ffmpeg-static**：自带 ffmpeg 二进制，把绿幕 MP4 烘焙成带透明通道的 WebM（VP9 / yuva420p），用户无需单独安装
- **Three.js 0.184**：WebGL 渲染，VideoTexture 直接采样透明 WebM 的 alpha 通道（非预乘 alpha，无需实时抠绿幕）
