# 宠物定制毛色与外观打磨设计

## 目标

使用用户上传的四张宠物照片，定制现有关节边境牧羊犬模型的毛色和花纹分布，同时细化模型表面，减少由低精度基础几何体拼接产生的机器人感。

## 范围

- 保留当前边境牧羊犬体型、骨架和自动坐下动作。
- 仅定制毛色、花纹位置和毛发表面效果。
- 提高模型网格精度，改善颈部、肩部、髋部、肘部、脚腕和脚掌的解剖过渡。
- 保留现有 2D 图片生成流程，并复用正面、左侧、右侧和全身四张照片。
- 仅在生成、重新生成或用户主动重新分析 3D 外观时调用视觉模型；启动软件时直接读取本地配置。

## OpenAI 外观分析

Responses API 支持一次输入多张图片，视觉模型能够分析颜色、形状和纹理；Structured Outputs 可通过 JSON Schema 约束返回结构。官方资料：

- https://developers.openai.com/api/docs/guides/images-vision
- https://developers.openai.com/api/docs/guides/structured-outputs

2D 图片生成成功后，发起一次独立视觉分析请求。请求将相同的四张照片作为 Base64 Data URL 输入，并设置 `detail: "high"`。默认使用 `gpt-5.4-mini`，在保持高精度视觉理解和结构化输出能力的同时控制成本。

提示词必须要求模型区分真实毛色与阴影、家具、项圈、玩具和背景颜色；对比四个视角，在照片互相冲突或区域被遮挡时降低置信度。

## 外观配置

模型通过严格 Schema 返回 `PetAppearanceProfile`：

- `version`：配置版本，初始为 `1`。
- `primaryColor`：主要毛色，标准六位十六进制颜色。
- `secondaryColor`：浅色花纹颜色。
- `accentColor`：可选第三毛色；不存在时等于主要毛色。
- `fur`：`short`、`medium` 或 `long`，仅影响表面粗糙度和轮廓柔化程度。
- `blaze`：额纹宽度、长度、水平偏移和边缘不规则度。
- `muzzle`：口鼻浅色覆盖率和左右不对称程度。
- `cheeks`：左右脸颊浅色覆盖率。
- `chest`：胸斑覆盖率和宽度。
- `belly`：腹部浅色覆盖率。
- `socks`：左前、右前、左后、右后四条腿的浅色袜高度。
- `bodyPatches`：最多四个身体椭圆斑块，包含侧面、纵向位置、垂直位置、宽、高、旋转和颜色角色。
- `tailTip`：尾尖浅色覆盖率。
- `confidence`：整体置信度，范围为 `0` 到 `1`。

本地代码必须限制所有数值范围并验证颜色格式。低置信度或未知区域回退到当前黑白边境牧羊犬默认值。

## 持久化与数据流

1. 设置窗口提交四张照片和图片模型。
2. 现有图片生成请求生成并保存 2D 宠物图片。
3. Responses API 视觉请求从相同照片提取 `PetAppearanceProfile`。
4. 主进程校验配置并保存到 `pet-config.json`。
5. 主进程向桌宠窗口发送 `pet-appearance-updated` 事件。
6. 渲染进程立即把配置应用到现有 Three.js 模型。
7. 后续启动时，`get-current-pet` 同时返回 2D 图片信息和已保存的 3D 外观配置。

如果 2D 图片生成成功但外观分析失败，生成图片仍然保存，之前有效的 3D 外观继续使用。设置窗口提示外观分析失败，并提供只重新分析 3D 外观的操作；该操作复用本地原图，不重新生成 2D 图片。

## 模型细化

保留现有关节层级，把明显有棱角的低面数几何体替换为更高精度网格，并启用平滑法线。肩部、髋部、颈部、肘部、脚腕和脚踝增加相互重叠的解剖过渡体。细化口鼻、眼睑、鼻子、耳朵和脚掌，同时保持桌面尺寸下的现有轮廓。

材质使用 `MeshPhysicalMaterial`，设置高粗糙度、低光泽和程序化法线贴图。本地 Canvas 噪声纹理提供具有方向性的毛发起伏，无需下载外部资源。毛长参数控制法线强度、粗糙度，以及躯干、颈部、脸颊和尾巴周围受限制的壳层轮廓效果，确保透明 Electron 窗口仍能流畅运行。

## 毛色渲染

根据校验后的配置，为头部、躯干、四条腿和尾巴生成本地 Canvas 纹理。每张纹理包含：

- 主要毛色底色。
- 额纹、口鼻、胸口、腹部、四肢和尾尖的柔边浅色遮罩。
- 带确定性边缘噪声的身体斑块。
- 低对比度方向性毛发纹理。

纹理使用固定种子的噪声，同一配置在重启后必须得到相同结果。花纹边缘需要柔和且略不规则。当前硬边白色花纹网格应删除或仅保留为轻微轮廓支撑，避免与纹理遮罩冲突。

## 界面

保留四张照片流程。生成期间依次显示“正在生成 2D 图片”和“正在分析 3D 毛色”两个阶段；完成后提示 2D 图片和个性化 3D 外观均已保存。

四张原图可用时显示 `重新分析 3D 外观` 按钮。该操作只调用视觉分析接口，不调用图片生成接口。

## 测试与验证

- 单元测试覆盖 Schema 标准化、数值限制、颜色验证和默认回退。
- 单元测试验证同一配置生成相同纹理描述。
- 集成测试验证外观配置持久化和 IPC 更新。
- 模型测试验证网格精度提升，并移除冲突的硬边花纹网格。
- Electron 截图至少渲染两套不同测试配置，验证同一体型出现不同毛色分布。
- 站立和坐下状态分别在 `260x250` 与 `180x180` 窗口验证画面非空、完整取景和关节动作不变。
- 性能采样确认透明 Electron 窗口内动画保持流畅。

## 验收标准

- 四张照片可以生成并保存经过校验的定制外观配置。
- 重启应用后恢复相同 3D 毛色，不再次调用 API。
- 重新生成或主动重新分析后更新毛色配置。
- 保留当前坐下动作和边境牧羊犬体型。
- 模型毛色和指定花纹来自配置，不再固定为黑白默认值。
- 更高精度表面和毛发材质明显减少机器人感。
- 分析失败时不删除最后一次有效的 2D 图片或 3D 外观配置。
