# Sora 桌面宠物动画设计

## 目标

以用户上传的四张宠物照片为身份参考，生成一张统一的标准坐姿图和三段写实动作视频。桌面宠物不再依赖简陋的程序化 3D 模型，而是播放与真实宠物外观一致的 Sora 视频动画。

## 范围

- 保留现有四张照片上传、OpenAI Platform API Key、本地加密保存和重新生成流程。
- 使用 GPT Image 生成一张标准坐姿参考图。
- 使用 `sora-2` 生成坐姿摇尾巴、站起再坐下、趴下睡觉再坐回三段视频。
- 使用纯色背景和 WebGL Shader 实时去背景。
- 视频生成后保存在本机，重启应用时直接复用，不再次调用 API。
- 当前程序化 Three.js 犬模型退出主显示流程；其源代码可在迁移验证完成后删除。
- 不再在生成流程中调用结构化毛色分析，避免与视频方案重复产生费用。

## 官方能力边界

Sora Videos API 支持图片参考作为视频首帧，并异步返回 MP4。图片参考必须与目标视频分辨率一致。API 不能严格指定最终帧，也不直接输出透明视频，因此本方案通过提示词要求动作回到坐姿，并在播放器端处理尾部过渡和背景透明化。

官方资料：

- https://developers.openai.com/api/docs/guides/video-generation
- https://developers.openai.com/api/reference/resources/videos

## 标准坐姿参考图

四张照片按正面、左侧、右侧和全身顺序输入现有 GPT Image 编辑接口。提示词要求：

- 忠实保留真实宠物的体型、脸型、耳朵、眼睛、毛长、毛色和所有可辨认花纹。
- 写实风格，不使用卡通、玩具、塑料或夸张幼态比例。
- 宠物自然坐姿，正面略偏三分之四角度，完整身体进入画面。
- 固定柔和棚拍光线和固定镜头。
- 使用单一纯色背景，无地面纹理、阴影、道具、文字或其他动物。

参考图生成后，本地等比缩放和留白到 `720x1280`，以降低生成成本并让宠物在桌面小窗口中占据主要画面。尺寸由 OpenAI Node SDK 当前 `VideoSize` 类型和请求校验测试共同锁定。

## 动画集合

所有视频使用相同标准坐姿图作为 `input_reference`，固定镜头、背景、光线和宠物大小。

### 坐姿待机

- 宠物保持坐姿，轻微呼吸、眨眼和自然摇尾巴。
- 动作幅度小，头部和脚掌位置稳定。
- 结尾回到起始坐姿。
- 播放器将该视频循环播放，作为所有动作之间的连接状态。
- 第一版时长为 `4` 秒。

### 站起再坐下

- 宠物从坐姿自然站起，短暂停留后重新坐下。
- 不行走，不改变朝向，不离开画面。
- 最后回到与首帧接近的标准坐姿。
- 第一版时长为 `8` 秒。

### 趴下睡觉再坐回

- 宠物从坐姿缓慢趴下，闭眼短暂停留，再自然起身坐回。
- 不翻滚，不改变镜头，不增加毯子或其他道具。
- 最后回到与首帧接近的标准坐姿。
- 第一版时长为 `8` 秒。

## Sora 请求与异步任务

主进程使用现有加密 API Key 和 Electron `net.fetch` 创建 OpenAI 客户端。每段视频创建独立任务：

1. 调用 `videos.create`，传入 `model: "sora-2"`、动作提示词、目标尺寸、时长和标准坐姿参考图。
2. 保存任务 ID、动作类型、状态和进度到本地配置。
3. 每 `10` 秒轮询一次 `videos.retrieve`，网络失败使用有上限的指数退避。
4. 状态为 `completed` 时调用 `videos.downloadContent` 下载 MP4。
5. 使用临时文件写入完成后原子替换正式文件，避免中断产生损坏视频。
6. 状态为 `failed` 时保留之前已成功的视频，并允许单独重试该动作。

应用关闭后不在后台继续轮询。重新启动时读取未完成任务 ID，并在用户打开设置窗口后恢复查询；不得重复创建新任务。

## 进度和费用控制

视频任务按顺序生成，不并发提交，避免突发费用和速率限制。界面分别显示：

- 正在生成标准坐姿图。
- 坐姿待机：排队、生成百分比、下载。
- 站起再坐下：排队、生成百分比、下载。
- 趴下睡觉：排队、生成百分比、下载。
- 本地资源已保存。

开始生成前明确提示将产生一次图片生成和三次视频生成费用。用户可以取消尚未提交的后续任务；已经提交到 OpenAI 的任务可能仍产生费用。

## 纯色背景与透明化

标准坐姿图和所有视频使用同一高饱和绿色背景 `#00ff00`。第一版先锁定该色键，避免不同动作使用不同背景造成边缘阈值不一致；后续再评估自动选择背景色。

桌宠窗口使用 `<video>` 作为 WebGL 纹理。片元 Shader 根据背景颜色计算色差、透明阈值和边缘羽化：

- 与背景颜色接近的像素 Alpha 设为 `0`。
- 毛发边缘使用平滑阈值，避免硬边。
- 对边缘残留色执行去色溢处理。
- 宠物主体颜色不得因去背景而明显改变。

如果 Shader 初始化失败，回退为普通视频并显示纯色背景，同时在设置窗口提示当前设备不支持透明处理。

## 动作播放器

播放器维护以下状态：

- `idle`：循环播放坐姿待机。
- `transition-out`：在待机循环的稳定帧停止，并淡入动作视频。
- `action`：播放站起或睡觉视频一次。
- `transition-in`：动作结束前短暂淡出，再回到待机视频。

动作结束时不能依赖 Sora 尾帧完全匹配。播放器在最后约 `150` 毫秒与待机首帧交叉淡化，降低姿态差异造成的跳变。随机动作只从已经下载成功的动画中选择；缺少动作视频时继续播放待机。

## 本地资源与持久化

用户数据目录保存：

- `canonical-sit.png`：标准坐姿参考图。
- `animation-idle.mp4`：坐姿待机。
- `animation-stand-sit.mp4`：站起再坐下。
- `animation-sleep.mp4`：趴下睡觉再坐回。
- 配置中的任务 ID、任务状态、背景颜色、模型、尺寸、时长和资源版本。

新一轮重新生成全部成功前，旧动画继续可用。单个新文件完成后先进入候选版本，三段视频和参考图全部成功后再一次性切换当前资源版本，避免混用不同宠物或不同生成批次。

## 错误处理

- `401`：提示 API Key 无效。
- `403`：提示项目没有 Videos 权限或 Sora 访问权限。
- `429`：保留任务状态并按退避时间重试查询，不重复创建任务。
- 网络超时：提示系统代理路径并允许继续等待。
- 任务失败：显示 OpenAI 返回的失败原因，只重试对应动作。
- 下载或文件写入失败：保留远端任务 ID，允许重新下载，不重新生成。
- 应用退出：停止本地轮询，不删除已完成文件和任务 ID。

## 测试与验证

- 单元测试覆盖动作提示词、任务状态机、轮询退避、取消后续任务和资源版本切换。
- 集成测试模拟 `videos.create/retrieve/downloadContent`，验证相同 API Key、`net.fetch`、任务恢复和错误映射。
- 测试保证标准坐姿图尺寸与视频请求尺寸完全一致。
- Shader 使用合成绿色背景视频帧验证透明像素、主体保留和边缘羽化。
- Electron 截图验证 `260x250` 和 `180x180` 窗口画面非空、无纯色背景残留和完整取景。
- 连续播放待机、站起、待机、睡觉、待机，检查切换期间没有明显空白帧或布局跳动。
- 在没有 Sora 权限的测试配置下验证清晰错误提示，且不破坏之前保存的宠物。

## 验收标准

- 使用同一 OpenAI Platform API Key 创建并下载三段 Sora 动画。
- 三段动画中的宠物外观与标准坐姿图保持可接受的一致性。
- 桌面显示中纯色背景被实时移除，宠物毛发边缘没有明显大块色边。
- 随机动作通过统一坐姿自然连接，切换时不出现空白窗口。
- 重启应用后直接使用本地视频，不产生 API 请求。
- 生成失败、网络中断或缺少 Videos 权限时保留旧宠物和旧动画。
