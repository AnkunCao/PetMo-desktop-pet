# AI Studio 设置窗口视觉设计

## 目标

将“创建我的桌面宠物”窗口升级为现代 AI 工具风格，同时完整保留照片上传、模型供应商、API Key、生成、动画进度和动作管理能力。

## 视觉方向

- 主色使用紫罗兰到靛蓝的渐变，搭配低饱和冷灰背景。
- 顶部采用深色渐变 Hero，明确产品名称、隐私状态和当前工作流。
- 内容卡片使用轻玻璃质感、细描边、柔和阴影和 16–20px 圆角。
- 主要操作按钮使用高对比渐变；次要操作保持安静。
- 避免大面积纯紫色和过度发光，确保长时间使用舒适。

## 信息架构

- 顶栏：产品标识、标题、简短说明、本地运行状态。
- 左栏：四视角照片上传，作为工作流第一步。
- 右栏上部：供应商、模型、API Key 和“记住 API”。
- 右栏中部：费用提示与生成/重新生成操作。
- 右栏下部：动画生成进度和动作管理入口。

桌面宽度保持双栏；窄窗口降为单栏。右栏信息密度高于左栏，但通过分组背景和间距保持可扫描性。

## 组件状态

- 上传卡片：默认、悬停、拖入、已有图片和更换状态。
- 表单控件：默认、聚焦、禁用和只读模型状态。
- 主按钮：默认、悬停、按下、禁用和生成中。
- 进度行：未开始、处理中、完成和失败。
- 状态提示：普通、成功、错误。
- 动作管理模态保持现有交互，视觉语言与主页面统一。

## 行为约束

- 不修改 IPC、生成流程、供应商切换或文件处理逻辑。
- 不改变现有 DOM ID 和测试依赖文案。
- 所有现有按钮、表单和模态交互继续工作。
- 支持 `prefers-reduced-motion`，减少不必要动效。

## 验收标准

- 主要任务“添加照片 → 配置模型 → 生成 → 查看进度”一眼可理解。
- 在 680px 最小宽度下无水平溢出或控件遮挡。
- 生成状态、错误状态和已完成状态有明确视觉区别。
- 所有现有自动化测试通过。
- Electron 实际渲染截图无裁切、重叠或不可读文字。
