# “记住 API”交互设计

## 目标

移除不符合生成流程的“保存配置”操作。用户选择供应商、输入 API Key 后可以直接生成；只有主动勾选“记住 API”时，密钥才会加密保存到本机。

## 界面

- 删除“保存配置”按钮。
- 在 API Key 输入区域增加“记住 API”复选框，默认不勾选。
- 供应商下拉框即时决定本次生成使用的图片与视频模型。
- 当前供应商存在已保存密钥时显示“已记住该供应商的 API Key”，但不把密钥回填到输入框。
- 切换供应商时清空输入框，并重新查询该供应商是否存在已保存密钥。

## 生成行为

生成请求必须携带：

```js
{
  photoDataUrls,
  apiKey,
  modelProvider,
  rememberApiKey
}
```

- 输入框有 Key：本次请求使用输入值。
- 输入框为空且当前供应商存在已保存 Key：使用已保存 Key。
- 两者都没有：在设置页显示缺少当前供应商密钥，不发起请求。
- `rememberApiKey: true`：在格式校验通过后，将 Key 按供应商加密保存。
- `rememberApiKey: false`：不保存新 Key；如果当前供应商以前有已保存 Key，则删除它。当前请求仍可使用本次输入值。
- 无论是否记住，输入框在生成请求完成或失败后清空。

## 密钥校验

API Key 在加密保存和构造 HTTP Header 前都执行同一套校验：

- 必须是字符串。
- 去除首尾普通空白后不能为空。
- 只允许可见 ASCII 字符，编码范围 `0x21` 到 `0x7E`。
- 禁止内部空格、换行、中文、全角字符和控制字符。
- OpenAI Key 继续要求 `sk-` 前缀与最短长度。
- Bytedance Key 接受火山方舟当前使用的可见 ASCII Key 格式，并保持最短长度校验。

校验失败时显示“API Key 包含中文、空格或换行，请只粘贴密钥本身”，请求不能进入网络层。

## 配置兼容

- 保留 `modelProvider` 和抠绿设置的自动持久化能力，不再由按钮触发。
- 密钥继续按 `encryptedApiKeys.openai` 与 `encryptedApiKeys.bytedance` 分开保存。
- 旧 `encryptedApiKey` 只兼容读取为 OpenAI Key。
- 删除已记住密钥时只删除当前供应商条目，不影响另一供应商。

## 测试

- 设置页不再存在“保存配置”按钮，并存在默认未勾选的“记住 API”。
- 生成请求携带 `rememberApiKey`。
- 未勾选时不保存新 Key。
- 未勾选且存在旧 Key 时删除当前供应商 Key。
- 勾选时只保存当前供应商 Key。
- 切换供应商会清空输入框并刷新已记住状态。
- 包含中文字符“自”、内部空格或换行的 Key 在网络调用前被拒绝。
- OpenAI 与 Bytedance 现有生成和动画测试保持通过。

## 验收标准

- 用户无需先执行“保存配置”即可生成。
- 默认情况下 API Key 不写入磁盘。
- 勾选“记住 API”后，下次启动可以留空使用相同供应商的已保存 Key。
- 取消勾选后生成会删除当前供应商已记住的 Key。
- 粘贴包含中文的 Key 只显示表单错误，不再出现 Electron 主进程异常弹窗。
